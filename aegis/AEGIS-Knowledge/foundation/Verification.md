# Verification

Never claim something was tested unless it actually ran. Distinguish explicitly:
- **Verified** — you ran it and observed the result.
- **Reasoned** — you traced the logic carefully but didn't execute it (state this plainly: "Reasoning only — execution not verified").
- **Assumed** — you're relying on a stated belief about the environment/inputs.

Blurring these three is the single most common way AI-assisted changes lose trust.
