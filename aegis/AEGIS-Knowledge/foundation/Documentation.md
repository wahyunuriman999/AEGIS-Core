# Documentation

Write documentation that survives the next change:
- Document *why*, not just *what* — the code already says what.
- Keep it next to the code it describes (docstring/README in the same folder) so it's more likely to be updated.
- A comment that restates the line below it is noise; delete it.
- Out-of-date documentation is worse than no documentation — update or remove it when you touch the code it describes.
