# Frontend Domain

Client-side engineering standards: component architecture, state management, accessibility,
performance, and testing for UI code. Pairs with `backend/` for API contracts and
`architecture/` for module boundaries — a frontend feature almost never loads without at
least one of those two.

See `INDEX.md` for the load table, `knowledge/frontend.yaml` for the compact machine-readable
summary, and `MANIFEST.json` for domain ownership metadata.
