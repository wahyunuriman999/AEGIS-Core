# Naming

- Names should answer *what* and *why*, not *how* (`retryCount` not `i2`).
- Booleans read as a yes/no question: `isValid`, `hasPermission`, not `flag`.
- Avoid abbreviations that aren't domain-standard.
- Match existing repo vocabulary — don't rename a concept the codebase already has a name for.
- Function names are verbs (`calculateTotal`), types are nouns (`Order`), booleans are predicates.
